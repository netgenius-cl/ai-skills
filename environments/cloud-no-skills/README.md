# Cloud No Skills

## Para que sirve

Esta guia es para agentes en la nube que no soportan skills persistentes ni archivos de instrucciones del proyecto.

Ejemplos:

- Gemini
- chats web de modelos generales
- asistentes que no pueden escribir archivos en el proyecto

## Como actuar

- no decir que dejaste algo instalado si no puedes hacerlo
- no pedir `git clone`
- no inventar requisitos previos
- no convertir esto en una guia de desarrollo
- usar modo simple por defecto en esta conversacion
- asumir que la persona no es tecnica

## Que hacer

1. Leer este catalogo.
2. Entender el comportamiento de `netgenius`.
3. Adoptar ese estilo en la conversacion actual.
4. Si el entorno no deja guardar instrucciones, decirlo simple.

## Respuesta recomendada

Si no hay persistencia:

`Aqui no puedo dejarlo instalado, pero ya te respondo simple y claro en esta conversacion.`

Si si puede guardar algun ajuste ligero:

`Listo, ya quedo configurado para responder facil y claro.`

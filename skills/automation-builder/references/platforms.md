# Platforms

Leer esta referencia solo si hay que elegir tecnologia o comparar varias.

## Comparacion rapida

- `n8n`: flexible y bueno para flujos mas complejos
- `Make`: visual y rapido para integraciones operativas
- `Zapier`: simple para conectar apps comunes con poca friccion
- `Power Automate`: fuerte en Microsoft 365, Teams, SharePoint y ecosistema empresarial
- `codigo`: mejor cuando hay requisitos muy propios o control tecnico profundo

## Criterios de eleccion

- cantidad de pasos
- complejidad de decisiones
- conectores ya disponibles
- necesidad de webhooks o APIs custom
- entorno donde vivira la automatizacion
- quien la mantendra despues

## Regla simple

- si el flujo es visual y flexible, pensar primero en `n8n`
- si es rapido y de apps comunes, pensar en `Make` o `Zapier`
- si vive en Microsoft, pensar en `Power Automate`
- si lo visual no alcanza, pensar en `codigo`

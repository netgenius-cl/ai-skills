# Navigation Checklist

Usar esta referencia cuando la tarea web sea larga, sensible o ambigua.

## Orientacion inicial

- URL actual
- titulo o encabezado principal
- cuenta o workspace visible
- pagina de lectura o pagina editable
- bloqueos visibles: login, modal, captcha, permiso, error

## Antes de hacer clic

- confirmar que el control correcto esta enfocado
- leer texto del boton y contexto inmediato
- validar si abre detalle, navega, guarda o ejecuta
- preferir un solo paso a la vez

## Antes de escribir

- identificar campo y etiqueta
- validar formato esperado
- revisar si hay autocompletado riesgoso
- confirmar que no se esta editando el registro equivocado

## Antes de enviar

- revisar resumen final
- confirmar impacto
- confirmar destinatario, archivo, monto o entidad objetivo
- verificar si existe forma de deshacer

## Senales de riesgo

- botones parecidos como Guardar, Guardar y publicar, Enviar, Confirmar
- dialogs pequenos con texto importante
- cambio de cuenta o workspace sin aviso
- formularios largos con autosave
- descargas o permisos no solicitados

## Regla de salida

Si el estado real de la pagina no es obvio, no adivinar.
